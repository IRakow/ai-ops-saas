Connect Your Repository

Give AI Ops read/write access to your codebase so it can open pull requests and push fixes on your behalf.

The platform that hosts your repository. This determines how we authenticate and open pull requests.

Full HTTPS clone URL for your repo — for example https://github.com/acme/backend. SSH URLs are not supported.

Personal Access Token (recommended): Create a fine-grained token with Contents: Read & Write and Pull Requests: Read & Write permissions.

Deploy Key: Paste your PEM-encoded private key (beginning with -----BEGIN). The corresponding public key must be added to your repo's deploy keys with write access.

Credentials are encrypted at rest using AES-256 and never logged.

The branch AI Ops will target for pull requests and pushes. Usually main or master.
